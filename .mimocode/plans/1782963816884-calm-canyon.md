# 健康提醒功能设计方案

## 背景

首页卡片"健康提醒"当前只是展示 `health_records` 记录条数，名不副实。需要一个真正的健康提醒系统，支持疫苗到期、用药结束、定期体检等提醒场景。同时修复 `/records` 路由缺失的问题。

## 一、数据库：新建 `health_reminders` 表

**Migration 文件**: `supabase/migrations/20260702000000_health_reminders.sql`

```sql
CREATE TABLE public.health_reminders (
  id                uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  pet_id            uuid NOT NULL REFERENCES public.pets(id) ON DELETE CASCADE,
  profile_id        uuid NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE,

  reminder_type     text NOT NULL CHECK (reminder_type IN ('vaccination', 'medication', 'checkup', 'custom')),
  title             text NOT NULL,
  description       text,
  due_date          date NOT NULL,

  -- 循环提醒
  repeat_interval   text CHECK (repeat_interval IN ('none', 'monthly', 'quarterly', 'yearly')),
  repeat_end_date   date,

  -- 状态
  is_completed      boolean NOT NULL DEFAULT false,
  completed_at      timestamptz,
  last_notified_at  timestamptz,

  metadata          jsonb DEFAULT '{}',
  created_at        timestamptz NOT NULL DEFAULT now(),
  updated_at        timestamptz NOT NULL DEFAULT now()
);

-- 索引：按宠物+到期日查询（首页卡片用，只查未完成的）
CREATE INDEX idx_health_reminders_due
  ON public.health_reminders(pet_id, due_date)
  WHERE is_completed = false;

-- 索引：按用户查询全部提醒
CREATE INDEX idx_health_reminders_profile
  ON public.health_reminders(profile_id, created_at DESC);

-- RLS
ALTER TABLE public.health_reminders ENABLE ROW LEVEL SECURITY;

CREATE POLICY health_reminders_read_auth
  ON public.health_reminders FOR SELECT
  USING (auth.uid() = profile_id);

CREATE POLICY health_reminders_manage_own
  ON public.health_reminders FOR ALL
  USING (auth.uid() = profile_id)
  WITH CHECK (auth.uid() = profile_id);

-- updated_at 自动更新
CREATE TRIGGER update_health_reminders_updated_at
  BEFORE UPDATE ON public.health_reminders
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

### reminder_type 说明

| 类型 | 用途 | 示例 title |
|------|------|-----------|
| `vaccination` | 疫苗到期提醒 | "猫三联加强针" |
| `medication` | 用药结束/复查提醒 | "驱虫药 - 大宠爱" |
| `checkup` | 定期体检提醒 | "年度体检" |
| `custom` | 用户自定义 | "剪指甲" |

## 三、后端 API（Next.js App Router）

复用 `web/app/api/pets/route.ts` 的模式：`getAuthUser` + `createAdminClient` + 手动校验 owner。

### 3.1 GET/POST — `web/app/api/health-reminders/route.ts`

**GET** `/api/health-reminders?petId=xxx&type=vaccination`
- 查询参数：`petId`（必填）、`type`（可选筛选）
- 返回该宠物的所有提醒，按 `due_date ASC` 排序

**POST** `/api/health-reminders`
- Body: `{ petId, reminderType, title, description?, dueDate, repeatInterval?, repeatEndDate? }`
- 服务端注入 `profile_id: user.id`

### 3.2 PATCH/DELETE — `web/app/api/health-reminders/[id]/route.ts`

**PATCH** — 更新提醒字段
**DELETE** — 删除提醒（校验 owner）

### 3.3 标记完成 — `web/app/api/health-reminders/[id]/complete/route.ts`

**POST** 标记完成逻辑：
1. 校验所有权
2. 设 `is_completed = true`, `completed_at = now()`
3. 如果 `repeat_interval != 'none'`：
   - 计算下一次 `due_date`（monthly +1月, quarterly +3月, yearly +1年）
   - 如果 `repeat_end_date` 存在且下次超过它，则不创建
   - 否则插入新提醒
4. 返回更新后的提醒 + 新创建的下一期（如有）

## 四、iOS 前端

### 4.1 新建 Composable — `ios-app-frontend/src/composables/useHealthReminders.js`

直接通过 Supabase 客户端操作（和现有 `useHealthRecords.js` 一致），不走 Next.js API。

功能：
- `fetchReminders(petId)` — 查询未完成提醒
- `addReminder(data)` — 创建提醒
- `completeReminder(id)` — 标记完成（循环提醒自动创建下一期）
- `deleteReminder(id)` — 删除
- `dueCount` — 即将到期的提醒数（首页卡片用）

### 4.2 新建页面 — `ios-app-frontend/src/views/HealthReminders.vue`

提醒列表页，展示当前宠物的所有健康提醒，支持：
- 按类型筛选（全部/疫苗/用药/体检/自定义）
- 标记完成
- 新建提醒
- 删除提醒

### 4.3 添加路由 — `ios-app-frontend/src/router/index.js`

```javascript
{ path: '/health-reminders', name: 'HealthReminders', component: () => import('../views/HealthReminders.vue') },
```

同时修复 `/records` 路由缺失：
```javascript
{ path: '/records', name: 'Records', component: () => import('../views/Records.vue') },
```

### 4.4 修改首页卡片 — `ios-app-frontend/src/views/Home.vue`

- 导入 `useHealthReminders`
- 将 `healthRecordCount` 替换为 `dueReminderCount`（即将到期的提醒数）
- 卡片点击跳转改为 `/health-reminders`
